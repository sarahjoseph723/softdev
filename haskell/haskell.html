Haskell

What is Haskell?

Haskell is a polymorphically (values can have more than one type). statically typed language. It utilizes lazy evaluation (expressions which are not needed to determine the answer to a problem aren't evaluated), and is purely functional, (it's evaluated like math). The syntax is mathematically derived, so it's pretty easy to pick up.

Pros and Cons

Pros

    Functional programs make it easier and cheaper to write large software (in terms of basic logic and testing, can then be translated)
    Smaller semantic gap between programmer and language
    Shorter, clearer, more maintainable, fewer errors, higher reliability
    Good for programs that need to be highly modifiable and maintainable
    Higher productivity

Cons

    Code tends to be slower than equivalent in other languages
    Difficult to debug (however, much less bugs to deal with)

        Functional Languages

        In functional languages, all the code is essentially treated like a single expression that gets evaluated. Order doesn't matter the same way it does in C or in Java (in languages like those if you put your functions in a different order you may get a different result, whereas in Haskell that doesn't happen). Functional languages like Haskell are strongly typed, so there are no core dumps.
