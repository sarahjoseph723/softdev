Syntax

The Basics

Haskell is case sensitive, utilizes types, has no side effects, and can take functions as parameters for other functions.

Tuples and Lists

Tuples can hold heterogeneous data types.

(5,3)
(5,3.0)
fst(5,3) will return 5

Haskell uses lists a lot. Unlike tuples, lists must be composed of homogenous data types.

ex = [1,2,3]

The colon is the cons operator (add an element to the begining of a list). All lists in Haskell are just representations of a bunch of cons operations for each element with an empty list. This is called syntactic sugar.

5:ex will return [5,1,2,3]

Strings

In Haskell, strings are just lists of characters.

word = 'h':'e':'l':'l':'o':[] will return "hello"

Lists and strings can be concatenated with the ++ operator.

word2 = "hello" ++ " world" will return "hello world"

Functions

Unlike many other languages, Haskell doesn't separate the parameters of its funcitons with parentheses.

square x = x * x
square 3 will return 9 square 5 will return 25

Functions in Haskell can take other functions as parameters.

test = [1,2,3,4,5]
foldr (+) 0 test --takes a function, initial value, and list as arguments
will return 15
0+1+2+3+4+5 = 15

Pattern Matching

A way to get values from other values using patterns.

let (a,b) = (10,12) in a * 2 will return 20
let (a:b:c:[]) = "xyz" in a will return 'x'
let (a:_:_:[]) = "xyz" in a will return 'x'
let (a:_) = "xyz" in a will return 'x'
